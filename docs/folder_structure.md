# MyMantra - Folder Structure

This document describes the complete folder structure for the MyMantra React Native application.

## Root Structure

```
myMantra/
├── android/                    # Android native code (auto-generated by React Native)
├── assets/                     # Static assets (images, fonts, audio, translations)
├── docs/                       # Documentation
├── ios/                        # iOS native code (auto-generated by React Native)
├── src/                        # Main source code
├── tests/                      # Test files
│   └── e2e/                    # End-to-end tests (optional)
└── [config files]              # Root-level configuration files
```

## Key Principles

### 1. **Separation of Concerns**
- **Components**: Reusable UI elements
- **Screens**: Full-screen views
- **Services**: Business logic and API calls
- **Store**: Global state management

### 2. **Feature-Based Organization**
- Screens organized by feature area (auth, mantra, goals, settings)
- Components organized by scope (common, mantra-specific, layout)

### 3. **Type Safety**
- Centralized TypeScript types in `src/types/`
- Shared across components, screens, and services

### 4. **Theming**
- All theme configuration in `src/theme/`
- 6 pre-defined color schemes
- Consistent spacing, typography, and shadows

### 5. **Assets at Root**
- Static resources separate from source code
- Easier for non-developers to manage
- Clear separation of code vs. media

## File Naming Conventions

### Components & Screens
- **PascalCase**: `MantraCard.tsx`, `LoginScreen.tsx`
- **Suffix**: `.tsx` for components, `.ts` for utilities

### Services & Utils
- **camelCase**: `googleAuth.ts`, `dateTime.ts`
- **Suffix**: `.ts` for non-React files

### Tests
- **Match source file**: `Button.test.tsx`, `validation.test.ts`
- **Suffix**: `.test.tsx` or `.test.ts`

### Types
- **camelCase**: `mantra.ts`, `goal.ts`, `user.ts`
- **Suffix**: `.ts`
