# MyMantra - Folder Structure

This document describes the complete folder structure for the MyMantra Flutter application.

---

## Root Structure

```
myMantra/
├── android/                    # Android native code (auto-generated by Flutter)
├── assets/                     # Static assets (images, fonts, audio, translations)
├── docs/                       # Documentation
├── ios/                        # iOS native code (auto-generated by Flutter)
├── lib/                        # Main Dart/Flutter source code
├── linux/                      # Linux desktop support (Phase 3.0+)
├── macos/                      # macOS native code (auto-generated by Flutter)
├── test/                       # Unit, widget, and integration tests
├── web/                        # Web platform files (auto-generated by Flutter)
├── windows/                    # Windows desktop support (Phase 3.0+)
├── pubspec.yaml                # Flutter dependencies & metadata
└── [config files]              # Root-level configuration (analysis_options.yaml, etc.)
```

## Key Principles

### 1. Clean Architecture
- **Domain layer** (`domain/`): Pure Dart entities, repository interfaces, use cases — zero framework dependencies
- **Data layer** (`data/`): Isar models, data sources, repository implementations
- **Presentation layer** (`presentation/`): Riverpod providers, screens, widgets

### 2. Feature-Based Organization
- Each feature is a self-contained module
- Cross-feature dependencies go through `shared/` or `core/`
- Facilitates independent testing and future team scaling

### 3. State Management (Riverpod)
- All state lives in Riverpod `providers/`
- Screens consume providers via `ConsumerWidget` or `ConsumerStatefulWidget`
- No global mutable state

### 4. Theming
- All theme configuration in `lib/app/theme/`
- Light and dark themes fully defined
- No hardcoded colors or text styles in widgets

### 5. Assets at Root
- Static resources separate from source code
- Clear separation of code vs. media

## File Naming Conventions

### Dart Files
- **snake_case**: `mantra_card.dart`, `session_screen.dart`, `create_mantra.dart`
- **Suffix guide**:
  - `_screen.dart` — full-screen widgets
  - `_widget.dart` — complex reusable widgets
  - `_provider.dart` — Riverpod providers
  - `_model.dart` — Isar-annotated data models
  - `_entity.dart` — Domain entities (or just the name: `mantra.dart`)
  - `_repository.dart` — Repository interfaces and implementations
  - `_usecase.dart` — Use case classes

### Classes
- **PascalCase**: `MantraCard`, `SessionScreen`, `MantraRepository`, `CreateMantra`

### Tests
- **Match source file + `_test`**: `mantra_card_test.dart`, `session_screen_test.dart`
- Located in `test/` mirroring the `lib/` structure
